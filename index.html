Sobre este Sistema Demonstrativo

O Sistema Integrado Edu foi desenvolvido apenas para fins demonstrativos e visualização de interfaces. Todos os sistemas são interativos, mas não implementam funcionalidades reais de backend ou integrações externas.

Você pode navegar livremente entre a Biblioteca, E-commerce e Banco Digital, interagir com os formulários e botões, mas todas as operações são simuladas para demonstrar os casos de uso na prática.

Os dados apresentados são fictícios e as transações não são reais.

Sistema de Biblioteca Online

Atores

  • Visitante
  • Usuário (cadastrado)
  • Sistema da Biblioteca

Falhas Possíveis

  • Visitante tenta emprestar sem cadastro
  • Tentativa de reservar livro disponível (reserva não permitida)
  • Renovação negada por atraso ou fila de espera

Casos de Uso

  • Buscar Livro:

    Visitante acessa, insere critérios e visualiza resultados.

  • Realizar Empréstimo:

    Usuário se autentica, escolhe livro e realiza empréstimo se disponível.

  • Reservar Livro:

    Sistema permite reserva apenas se o livro estiver emprestado.

  • Renovar Empréstimo:

    Sistema renova se não houver atraso ou fila de espera.

Sistema Bancário

Atores

  • Usuário
  • Empresa
  • Sistema de Transferência
  • Sistema de Pagamento
  • Gerente

Falhas Possíveis

  • Transferência falha por saldo ou dados incorretos
  • Pagamento de salário com erro
  • Conta bloqueada indevidamente
  • Empréstimo negado por restrições

Casos de Uso

  • Entrar na Conta:

    Usuário informa dados e acessa se autenticado.

  • Transferência:

    Usuário envia valor, sistema valida e transfere (com gerente, se necessário).

  • Salário:

    Empresa agenda, sistema transfere e usuário recebe.

  • Empréstimo:

    Usuário solicita, sistema avalia e aprova ou recusa.

  • Contratar Seguro:

    Usuário escolhe seguro, aceita termos e sistema registra.

Sistema de E-commerce

Atores

  • Cliente
  • Sistema de Pagamento
  • Sistema de Entrega
  • Sistema de Cupons

Falhas Possíveis

  • Cupom inválido
  • Pagamento recusado
  • Produto fora de estoque
  • Erro no frete

Casos de Uso

  • Buscar Produto:

    Cliente insere critérios e recebe resultados.

  • Adicionar ao Carrinho:

    Produto selecionado e adicionado.

  • Finalizar Compra:

    Cliente informa entrega e pagamento, aplica cupom (opcional) e finaliza pedido.
